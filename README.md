## Summary

A multimodal AI pipeline that turns a single product image into a publish-ready Product Detail Page (PDP) in English and Gulf Arabic — in under 10 seconds. Built for Mumzworld's catalog operations team: 4 chained agents handle vision extraction, English copy, Arabic copy (native Gulf register, not a translation), and a self-critique pass that catches hallucinations before output. Every field carries a confidence score; anything below the threshold is nulled rather than guessed. A `review_required` flag is raised automatically when confidence is low or the critique finds errors, keeping a human in the loop for uncertain cases.

## What It Does

```
Product Image
      │
      ▼
[Agent 1: Vision Extraction]      ← multimodal vision via Groq (Llama-4-Scout)
  Extracts: product_type, color, brand, visible_features,
            age_indicators, material_hints
  Tags each field: "visible" | "inferred" | "uncertain"
      │
      ▼
[Knowledge Base Lookup]           ← in-memory RAG (12 categories)
  Fetches: safety standards, age guidelines, Arabic naming conventions
      │
      ▼
[Agent 2a: English PDP Generator] ← separate generation pass
  Outputs: title (≤70 chars), description, key_features (≤7),
           safety_warnings, age_suitability, search_tags
      │
      ▼
[Agent 2b: Arabic PDP Generator]  ← NOT a translation — native Gulf Arabic
  Same inputs as EN, separate system prompt in Arabic
  Self-scores Arabic nativeness 1–5
      │
      ▼
[Agent 3: Self-Critique Loop]     ← temperature=0, deterministic
  Checks: grounding (no hallucinations), Arabic quality, safety completeness
  Returns: CritiqueFinding list + revised confidences
      │
      ▼
[Confidence Gate]
  Fields < 0.40 confidence → None (never a hallucinated value)
  overall_confidence < 0.70 OR any error → review_required = True
      │
      ▼
PDPResult (Pydantic-validated JSON)
  ├── content_en: PDPContent
  ├── content_ar: PDPContent
  ├── field_confidences: list[FieldConfidence]
  ├── overall_confidence: float
  ├── review_required: bool
  └── critique_notes: list[CritiqueFinding]
```

## Tradeoffs

See [TRADEOFFS.md](TRADEOFFS.md) for full architecture decisions.

| Decision | Choice | Rejected Alternative | Reason |
|---|---|---|---|
| Model | Groq free API (Llama-4-Scout-17B) | Gemini API, NVIDIA NIM | Free, fast (~5s/run), OpenAI-compatible SDK |
| Knowledge base | In-memory JSON | pgvector + embeddings | Honest 5-hour scope |
| Arabic approach | Separate generation pass | Translation of EN | Native copy converts better |
| Structured output | JSON mode + Pydantic | response_schema param | More reliable across model versions |
| UI | Streamlit | FastAPI + React | Fastest path to runnable demo |
| Self-critique | Separate agent pass | Single-pass | Catches hallucinations before output |

## Tooling

| Tool | Role |
|---|---|
| **Claude (via Antigravity)** | Primary development harness — architecture design, pair-coding, agent prompt engineering, schema design, Streamlit scaffolding |
| **Groq** (free tier) | Primary model backend — `meta-llama/llama-4-scout-17b-16e-instruct` for all 4 AI passes, ~5s/run |
| **NVIDIA NIM / OpenRouter** | Fallback backends — automatic failover if `NVIDIA_API_KEY` or `OPENROUTER_API_KEY` is set |
| **OpenAI Python SDK** | API client — same code works against Groq, NVIDIA NIM, and OpenRouter (all OpenAI-compatible) |
| **Pydantic v2** | Schema enforcement — every pipeline output validated, failures explicit not silent |
| **Streamlit** | Demo UI — fastest path to a runnable bilingual interface |
| **structlog** | Structured logging with correlation IDs across all pipeline passes |
| **PIL / Pillow** | Image loading for vision agent + synthetic test image generation |

**How AI was used:**
- Claude / Antigravity generated the project structure, Pydantic schemas, agent code skeletons, Streamlit UI layout, and eval harness structure
- Arabic system prompts required significant manual iteration — the first-pass prompt produced translated English. Fixed by embedding Gulf Arabic examples and explicit anti-translation instructions
- Confidence gate thresholds (0.70 / 0.40) were calibrated manually after initial eval runs showed over-flagging at 0.80
- All 15 test case labels and expected values were written by hand

**Where I overruled the agent:**
- Arabic `self_critique` prompt: initial version was too lenient. Added explicit "be strict, 3 is already acceptable" instruction
- Knowledge base Arabic content: verified and corrected Arabic phrasing for 3 categories where the model generated awkward Gulf Arabic
- `confidence_gate.py` weight distribution: agent's initial 50/50 EN/AR split changed to 90% content / 10% Arabic since a weak AR draft shouldn't tank an otherwise strong EN PDP
